ChatGPT · 15. Juli

Custom Instructions jetzt bis 5.000 Zeichen

Was ist das? „Custom Instructions" ist ein dauerhaftes Textfeld in den ChatGPT-Einstellungen, das ChatGPT vor jeder Antwort liest — quasi ein Steckbrief über dich und deine Wünsche (wer du bist, welchen Ton du willst, was ChatGPT tun und lassen soll). Bisher war es auf 1.500 Zeichen begrenzt, jetzt sind es 5.000.

Warum relevant: Mehr Platz heißt, du kannst ein richtiges kleines Regelwerk hinterlegen statt nur Stichworte — das ist einer der größten Qualitätshebel überhaupt, weil es bei jeder Unterhaltung greift, ohne dass du es wiederholen musst.

So nutzt du's: Einstellungen → Personalisierung → Custom Instructions. Schreib in Feld 1, wer du bist und woran du arbeitest; in Feld 2 den gewünschten Stil (z. B. „antworte auf Deutsch, kurz, mit konkreten Beispielen, keine Floskeln"). Am besten als Do-/Don't-Liste.

So nutzt du's: Bei einer Seiten-Frage einfach /fork tippen — die Untersuchung läuft separat weiter. An einem sinnvollen Zwischenstand gezielt /code-review aufrufen.

MCP · finalisiert 28. Juli

MCP wird „stateless" — und bekommt Apps

Was ist das? MCP (Model Context Protocol) ist ein offener Standard, über den KI-Assistenten an externe Werkzeuge und Datenquellen andocken — der „USB-Anschluss" zwischen KI und deinen Tools. Bisher musste ein Client beim Verbinden erst einen Handshake machen und eine feste Sitzung halten. „Stateless" heißt: Dieser Zwang fällt weg, jede Anfrage kann an jeden Server-Knoten gehen. Zusätzlich neu: „MCP Apps" — Server dürfen kleine interaktive Oberflächen anzeigen.

Warum relevant: Für alle, die eigene MCP-Server betreiben, wird das Skalieren viel einfacher (kein spezieller Sitzungs-Server nötig). Mit „MCP Apps" werden Tools von reinen Textantworten zu echten kleinen Bedienoberflächen.

So nutzt du's: Wenn du einen MCP-Server hast, auf die neue Spec umsteigen und ihn hinter einen einfachen Load-Balancer stellen. Achtung: Wer die experimentelle „Tasks"-Schnittstelle nutzt, muss anpassen (Breaking Change).
Quelle: modelcontextprotocol.io
Cursor · 2.–17. Juli

Automations, Side Chats & Slack

Was ist das? Cursor ist ein Code-Editor mit eingebautem KI-Assistenten. „Automations" starten einen Agenten automatisch, wenn ein Auslöser eintritt — etwa ein neuer Commit, eine Slack-Nachricht oder ein Zeitplan. „Side Chats" sind Neben-Unterhaltungen, die parallel zum Haupt-Agenten laufen.

Warum relevant: Damit lassen sich wiederkehrende Entwickler-Aufgaben automatisieren (ähnlich einer CI-Pipeline), ohne dass du jedes Mal selbst anstößt.

So nutzt du's: Eine Automation einrichten, die z. B. bei jedem Commit einen Test- oder Review-Agenten laufen lässt; für Experimente Side Chats parallel zum Hauptstrang nutzen.
Quelle: cursor.com/blog
Desktop Commander · MCP-Tool

Desktop Commander MCP

Was ist das? Ein MCP-Server (siehe oben), der einem KI-Assistenten Zugriff auf dein ganzes Betriebssystem gibt — Terminal-Befehle ausführen, Dateien im gesamten Rechner suchen und gezielt per „Suchen-und-Ersetzen" bearbeiten. Also nicht nur innerhalb eines Editors, sondern systemweit.

Warum relevant: Damit kann Claude echte Arbeit auf deinem Rechner erledigen — Skripte laufen lassen, Dateien umbauen, Tabellen (CSV/Excel) verarbeiten. Eines der aktuell meistgenutzten MCP-Tools (~6,3k Sterne).

So nutzt du's: Den Server nach Anleitung installieren und in Claude als MCP-Verbindung eintragen; danach kann der Assistent Befehle und Dateiänderungen ausführen (bewusst und mit Bedacht einsetzen).

OpenManus — Baukasten für autonome Agenten

Was ist das? Ein quelloffenes Framework (kostenlos, MIT-Lizenz), mit dem man eigene „autonome Agenten" baut — KI-Programme, die selbstständig mehrschrittige Aufgaben abarbeiten, etwa im Web recherchieren oder Daten analysieren. Entstanden aus dem Team hinter MetaGPT.

Warum relevant: Wer selbst experimentieren will (statt fertige Produkte zu nutzen), hat hier eine breit genutzte, gut gepflegte Grundlage — inklusive einer Variante mit „Reinforcement Learning".

Wofür du's nutzt: Eigene Agenten für Web-Automation, Datenanalyse oder Multi-Agent-Workflows aufsetzen und anpassen.
Quelle: github.com/FoundationAgents/OpenManus

chrome-devtools-mcp — Browser-Debugging für Agenten

Was ist das? Ein offizielles Google-Repo, das die Chrome-Entwicklerwerkzeuge als MCP-Server bereitstellt. Damit bekommt ein KI-Agent echten Zugriff auf einen Browser: Seitenaufbau (DOM), Netzwerk und Konsole inspizieren, Aktionen automatisieren.
